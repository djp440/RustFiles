# RustFiles V1 CDD 规格文档

## 1. 需求摘要

- 需求目标：将已批准的 RustFiles V1 PRD 收敛为架构设计前的规格文档，明确后续必须做对的行为、风险等级、验证路径、测试数据隔离和完成边界。
- 交付对象：面向 Windows 10 / 11 的本地文件管理器 V1，技术假设为 Rust 核心、React UI、Tauri 桌面壳。
- 核心闭环：打开应用 -> 进入文件夹 -> 多标签、路径栏、面包屑和侧边栏导航 -> 快速浏览 -> 搜索、过滤、排序 -> 复制、移动、删除、重命名 -> 稳定完成。
- 关键约束：V1 聚焦本地文件管理；必须支持基础多标签页；必须遵循 UI 优先调度；必须避免后台扫描、缩略图、搜索和文件操作拖慢前台体感。
- 明确非目标：不做云盘同步、远程协议、插件、全文索引、AI 整理、压缩包浏览、内置终端、内置预览器、复杂标签页工作区或跨设备同步。

## 2. 流程位置

- 当前阶段：需求分析之后，架构设计之前。
- 上游文档：`docs/cdd-brainstorming/prds/2026-05-21-rustfiles-v1-prd.md`。
- 下一阶段：`cdd-architecture-design`。
- 该文档将服务于：架构设计、实现计划、测试计划、验收测试和发布前风险审查。
- 本文档不包含架构方案、模块拆分、API 设计或实现任务派发。

## 3. 验证目标

### 3.1 必须保证的行为

- 文件浏览：本地磁盘、常用位置、目录进入/返回/前进、地址栏跳转、面包屑导航和三种文件视图可用。
- 多标签页：标签页可新建、关闭、切换，并能独立维护路径、历史、视图、排序和搜索状态。
- 搜索与过滤：当前目录文件名搜索实时响应，递归搜索可开关，搜索结果可打开所在位置。
- 文件操作：新建文件夹、重命名、删除到回收站、永久删除、复制、剪切、粘贴、拖拽移动、拖拽复制、多选、右键菜单、默认应用打开、终端打开当前目录和属性查看可用。
- 冲突处理：复制/移动冲突支持替换、跳过、保留两者、应用于全部。
- 缩略图与图标：图片缩略图、文件夹图标和常见文件类型图标可异步加载，并优先服务可视区域。
- 性能体感：当前可见 UI 的输入响应、滚动、动画、视图切换和路径导航优先级高于后台任务。
- 设置：默认启动路径、默认视图、隐藏文件、扩展名、主题、排序偏好、缩略图和动效设置可保存并生效。

### 3.2 允许退化但必须说明的行为

- 后台扫描、递归搜索、缩略图生成、复制和移动任务可以在用户交互时降速。
- 大目录中非可视区域的缩略图可以延迟生成。
- 视频缩略图和 PDF 缩略图可以不进入 V1 完成声明，只作为 V1.1 候选。
- 性能指标在明显慢速磁盘、网络挂载路径、系统杀毒软件强干扰或极端文件权限环境下可以单独标记为受限，不得并入普通通过结论。

## 4. 运行时环境判断

- 环境：`hybrid`，包含 `desktop`、`web UI`、`Rust 后台/文件系统` 和 `Windows 系统集成`。
- 当前仓库状态：目前只有文档，没有 Tauri、Rust、React 运行时或测试脚手架。
- 当前可用验证能力：
  - 可读写仓库文档。
  - 可执行 shell 命令。
  - 可通过 git diff 和状态检查确认文档变更。
  - 后续一旦有前端运行时，可使用浏览器类工具进行 UI 验收。
  - 后续一旦有桌面运行时，可通过本地进程、截图、日志和文件系统副作用验证。
- 当前不可用验证能力：
  - 无法验证真实启动速度、目录打开速度、滚动帧率或 1 万文件目录可交互时间。
  - 无法验证真实 Windows 回收站、默认应用打开、终端打开、属性弹窗和拖拽行为。
  - 无法验证 Tauri 打包、自动更新、崩溃日志或安装包。
- 声明边界：当前只能声明 CDD 规格文档已定义验证目标与完成边界，不能声明产品功能或性能已经实现。

## 5. 风险等级与升级条件

- 等级：`R4`。
- 原因：V1 涉及永久删除、替换、移动、剪切、粘贴、批量操作、拖拽移动和文件冲突处理，存在破坏用户本地数据的风险；同时涉及桌面系统集成、回收站、默认应用、终端打开和本地缓存，错误实现会损害用户信任。
- 是否必须人工审核：是。所有破坏性文件操作、冲突处理、拖拽语义、回收站集成、永久删除确认和任务恢复策略都必须经过人工审核。
- 停止点或升级条件：
  - 任何可能绕过回收站、误删真实用户目录、覆盖原文件或跨卷移动失败后状态不一致的实现，必须停止并人工审查。
  - 任何无法说明任务失败后文件源/目标状态的复制、移动、剪切、粘贴实现，必须停止。
  - 任何性能优化会牺牲文件操作正确性、错误可见性或数据完整性的方案，必须停止。
  - 任何需要管理员权限、修改系统级配置、注册文件关联或影响系统资源管理器行为的方案，必须升级审查。
  - 任何测试需要触碰用户真实桌面、下载、文档、图片、视频或音乐目录时，必须改用隔离夹具目录。

## 6. 开发阶段建议

后续开发应沿用 PRD 的细粒度版本节奏，每个版本集中验证同一类能力：

1. V0.1 浏览骨架。
2. V0.2 视图与排序。
3. V0.3 性能底座。
4. V0.4 搜索。
5. V0.5 文件操作基础。
6. V0.6 复制移动闭环。
7. V0.7 冲突处理与多选。
8. V0.8 多标签页。
9. V0.9 缩略图与图标。
10. V1.0 商业首发打磨。

先做的保护性检查：

- 在任何文件写入/删除/移动功能之前，先定义统一的测试根目录和路径安全规则。
- 在永久删除之前，先完成删除到回收站和二次确认验证。
- 在复制/移动冲突处理之前，先完成任务状态模型和可恢复错误模型。
- 在缩略图缓存之前，先定义缓存目录、失效策略和清理策略。
- 在性能指标验收之前，先定义可复现的大目录夹具和测量方法。

需要避免的捷径：

- 不得直接在真实用户目录中做破坏性测试。
- 不得把模拟文件操作成功当成真实文件系统成功。
- 不得用同步阻塞调用换取短期实现简单。
- 不得在 UI 层直接决定不可逆文件操作。
- 不得缺少冲突处理和失败恢复就声称复制/移动闭环完成。

## 7. 验证计划

### 7.1 静态检查

后续实现阶段至少需要：

- Rust：格式化、lint、单元测试和关键文件操作逻辑测试。
- TypeScript/React：类型检查、lint、组件测试或聚焦行为测试。
- Tauri 配置：权限、命令暴露、打包配置和平台目标检查。
- 文档：PRD、CDD、架构设计和实现计划中的范围一致性检查。

### 7.2 运行时验证

必须覆盖：

- 启动应用并进入默认目录。
- 侧边栏、地址栏、面包屑、前进/后退和多标签导航。
- 普通目录浏览、排序、过滤和视图切换。
- 1 万文件目录初次渲染、滚动和交互。
- 当前目录搜索、递归搜索和打开所在位置。
- 图片缩略图异步加载、缓存命中和滚动时延迟加载。
- 新建文件夹、重命名、删除到回收站、永久删除。
- 复制、剪切、粘贴、拖拽移动、拖拽复制。
- 冲突处理四路径：替换、跳过、保留两者、应用于全部。
- 后台任务运行中前台 UI 的滚动、选择、右键菜单、路径跳转和标签切换。
- 浅色、深色、跟随系统主题与基础设置持久化。

### 7.3 负面路径

必须覆盖：

- 目标文件已存在。
- 文件被占用或权限不足。
- 源路径消失。
- 目标路径消失。
- 跨磁盘移动失败。
- 操作中断或应用关闭。
- 永久删除取消。
- 回收站不可用或失败。
- 递归搜索遇到无权限目录。
- 缩略图生成失败或文件损坏。
- 缓存目录不可写。
- 大目录快速滚动时取消非可视缩略图任务。
- 多标签页中后台标签任务失败，不污染当前标签状态。

### 7.4 证据收集方式

- 自动化测试结果：命令输出、退出码、测试报告。
- 桌面运行证据：截图、录屏或操作日志。
- 文件系统断言：源目录、目标目录、回收站路径或隔离夹具目录中的文件状态。
- 性能证据：启动时间、目录打开时间、大目录首屏可交互时间、滚动帧率或帧时间、后台任务期间交互延迟。
- 崩溃与错误证据：日志、错误消息、任务失败状态和恢复结果。

## 8. 测试数据与状态隔离

### 8.1 输入或夹具

必须准备隔离测试根目录，例如：

- `test-fixtures/small-dir`：少量文件和文件夹。
- `test-fixtures/large-10k-dir`：1 万个文件，覆盖不同扩展名、大小和修改时间。
- `test-fixtures/media-dir`：图片文件、损坏图片、非图片伪装扩展名。
- `test-fixtures/conflict-source` 与 `test-fixtures/conflict-target`：用于复制/移动冲突。
- `test-fixtures/permission-cases`：只读文件、占用文件、无权限目录。
- `test-fixtures/deep-tree`：多层目录和递归搜索数据。

### 8.2 设置

- 所有破坏性测试必须在临时目录或明确的测试根目录中执行。
- 测试运行前必须清空并重建夹具目录。
- 文件名应覆盖中文、空格、长文件名、大小写差异、特殊扩展名和无扩展名。
- 时间和大小排序测试必须固定文件时间戳和文件大小。

### 8.3 清理

- 测试结束后必须删除临时目录、缩略图测试缓存和任务状态测试文件。
- 如果验证回收站行为，必须使用专门生成的测试文件，并记录清理步骤。
- 不允许依赖手动清理真实用户目录。

### 8.4 隔离策略

- 禁止对真实桌面、下载、文档、图片、视频、音乐目录执行删除、移动或覆盖测试。
- 默认应用打开和终端打开可以使用无害测试文件和隔离目录。
- 缩略图缓存必须可配置到测试专用缓存目录。
- 多标签页状态测试必须使用独立夹具目录，避免标签之间共享脏状态。

## 9. Mock / Sandbox 边界

- 可以使用 mock 或 sandbox 验证：
  - UI 状态流转。
  - 多标签页状态隔离。
  - 冲突处理弹窗选择逻辑。
  - 任务队列状态机。
  - 缩略图加载占位、取消和优先级排序。
- 必须使用真实文件系统验证：
  - 新建、重命名、复制、剪切、粘贴、删除、移动和冲突处理的最终文件状态。
  - 回收站集成。
  - 默认应用打开。
  - 终端打开当前目录。
  - 权限不足、文件占用、路径消失等错误路径。
- 仍可证明的内容：mock 可以证明交互状态、分支选择和 UI 反馈逻辑。
- 不可声称的内容：mock 不能证明真实 Windows 文件系统操作、真实回收站行为、真实性能、真实拖拽系统集成或真实崩溃率。

## 10. 完成标准

### 10.1 可以声明完成

只有满足以下条件时，才可以声明对应 V1 能力完成：

- 功能范围与 PRD 和本 CDD 文档一致。
- 相关静态检查通过。
- 相关运行时验证通过。
- 负面路径至少覆盖本 CDD 标记的必测场景。
- 文件操作在隔离真实文件系统夹具中验证最终状态。
- 性能指标有可复现测量证据。
- UI 优先调度在后台任务压力下有运行时证据。
- 高风险文件操作经过人工审核。

### 10.2 不可以声明完成

不得在以下情况下声明完成：

- 只通过 mock，没有真实文件系统验证。
- 只验证成功路径，没有验证冲突、权限、占用、取消或失败恢复。
- 只在小目录下验证，却声称大目录性能达标。
- 只验证前台空闲状态，却声称后台任务不影响 UI。
- 没有测试数据隔离，或测试依赖真实用户目录。
- 没有人工审核破坏性文件操作。

### 10.3 仍需人工确认

以下内容必须在进入实现计划和发布前分别人工确认：

- 永久删除、覆盖、剪切、移动和冲突处理的安全语义。
- UI 优先调度是否符合用户体感。
- Windows 10 / 11 上的回收站、默认应用、终端打开和拖拽行为。
- V1.0 商业首发是否满足“可日常替代资源管理器”的主观体验标准。

## 11. 明确不声称的内容

当前 CDD 阶段不声称：

- RustFiles 已实现任何产品功能。
- 性能指标已经达到 PRD 数值。
- Windows 系统集成已经可用。
- 文件操作已经安全。
- UI 设计已经定稿。
- 架构方案已经确定。
- V1.0 已具备商业发布质量。

本文档只声明：V1 的风险、验证目标、测试数据隔离、mock 边界和完成标准已经被规格化，可交给下一阶段进行架构设计。
