# RustFiles V1 PRD

## 1. 产品定位

RustFiles 是一款面向 Windows 10 / 11 用户的高性能、精致、现代化本地文件管理器。

V1 的主方向是“漂亮顺手的日常替代品”：让用户愿意在日常文件管理中替代 Windows 资源管理器。性能与稳定是硬底线，UI 体验是差异化卖点。

关键词：快、稳、漂亮、统一、顺手、可替代资源管理器。

## 2. 目标用户

V1 面向 Windows 本地文件高频用户，包括：

- 开发者：经常在项目目录、下载目录、终端和编辑器之间切换。
- 内容创作者：经常管理图片、视频、素材和导出文件。
- 办公用户：经常处理文档、下载文件、桌面文件和多层文件夹。
- 大量文件管理用户：需要浏览、搜索、排序和批量操作大量文件。

V1 不优先服务企业协作、云盘同步、远程文件协议或高级数字资产管理场景。

## 3. 核心目标

V1 只服务一个最小闭环：

用户打开应用 -> 进入文件夹 -> 通过多标签、路径栏、面包屑和侧边栏导航 -> 快速浏览 -> 搜索、过滤、排序 -> 复制、移动、删除、重命名 -> 稳定完成。

核心目标：

1. 打开目录极快，普通文件夹近乎瞬开，大目录尽快可交互。
2. 文件浏览体验优雅，视图、排序、过滤、路径导航都顺手。
3. 基础文件操作稳定可靠，有明确进度、错误反馈和冲突处理。
4. UI 明显比 Windows 资源管理器更现代、更统一，但不为了炫技牺牲效率。

## 4. V1 必做范围

### 4.1 文件浏览

必须支持：

- 本地磁盘浏览。
- 文件夹进入、返回、前进。
- 地址栏路径跳转。
- 面包屑路径导航。
- 侧边栏入口：
  - 快速访问。
  - 桌面。
  - 下载。
  - 文档。
  - 图片。
  - 视频。
  - 音乐。
  - 此电脑。
  - 各磁盘分区。
- 文件列表视图：
  - 图标视图。
  - 列表视图。
  - 详细信息视图。
- 排序：
  - 名称。
  - 修改时间。
  - 大小。
  - 类型。
- 基础过滤：
  - 文件。
  - 文件夹。
  - 图片。
  - 视频。
  - 文档。

### 4.2 多标签页

V1 必须支持基础多标签页：

- 新建标签页。
- 关闭标签页。
- 切换标签页。
- 从文件夹右键在新标签页打开。
- 每个标签页维护独立路径、历史前进/后退、视图模式、排序状态和搜索状态。

V1 不要求标签分组、保存会话、跨窗口拖拽标签、标签页同步或复杂工作区恢复。

### 4.3 文件操作

V1 必须稳定支持：

- 新建文件夹。
- 重命名。
- 删除到回收站。
- 永久删除。
- 复制。
- 剪切。
- 粘贴。
- 拖拽移动。
- 拖拽复制。
- 多选。
- 右键菜单。
- 打开文件。
- 在默认应用中打开。
- 在终端中打开当前目录。
- 显示属性。

文件冲突处理必须支持：

- 替换。
- 跳过。
- 保留两者。
- 应用于全部。

### 4.4 搜索

V1 只做当前目录搜索，不做全盘复杂索引。

必须支持：

- 当前目录文件名搜索。
- 实时过滤。
- 递归搜索开关。
- 搜索结果可打开所在位置。

不做：

- 全文搜索。
- OCR 搜索。
- 云盘搜索。
- 复杂标签系统。

### 4.5 缩略图与图标

V1 必须支持：

- 图片缩略图。
- 文件夹图标。
- 常见文件类型图标。

缩略图策略：

- 异步生成。
- 本地缓存。
- 优先加载可视区域。
- 滚动时延迟加载，避免影响滚动流畅性。

视频缩略图和 PDF 缩略图可放入 V1.1 候选，不作为 V1 商业首发阻断项。

### 4.6 设置

V1 只做基础设置：

- 默认启动路径。
- 默认视图模式。
- 是否显示隐藏文件。
- 是否显示文件扩展名。
- 主题：
  - 跟随系统。
  - 浅色。
  - 深色。
- 文件排序偏好。
- 缩略图开关。
- 动效开关。

## 5. 性能与调度原则

性能是 RustFiles 的核心卖点。V1 的性能目标必须围绕用户体感，而不只是围绕后台任务吞吐量。

### 5.1 UI 优先调度

系统资源调度必须优先保障当前可见 UI 的输入响应、滚动、动画、视图切换和路径导航。

文件扫描、缩略图生成、递归搜索、复制、移动、删除等 IO 密集任务都应作为后台任务分级调度。在资源紧张或用户正在交互时，后台任务必须主动让出资源。

验收口径：

- 后台正在扫描大目录时，当前标签页仍可顺畅滚动、选择和右键操作。
- 后台正在生成缩略图时，当前可视列表优先加载，非可视区域延后。
- 后台正在复制或移动文件时，前台路径跳转、标签切换和视图切换不应卡顿。
- 后台任务可以变慢，但 UI 不应假死或明显掉帧。

### 5.2 性能指标

V1 建议指标：

| 指标 | 标准 |
| --- | --- |
| 冷启动 | <= 800ms |
| 普通文件夹打开 | <= 100ms |
| 1 万文件目录初次渲染 | <= 500ms 可交互 |
| 列表滚动 | 稳定 60fps |
| 文件操作 | 不阻塞 UI |
| 大目录浏览 | 必须使用虚拟列表 |
| 缩略图加载 | 异步加载，优先可视区域 |
| 搜索/扫描 | 不阻塞主线程 |

## 6. UI 与体验要求

V1 UI 定位：现代、精致、克制、高密度但不拥挤。

主界面应像一个长期使用的生产力工具，而不是营销页面或展示型 Demo。界面层级必须清楚：

- 标签栏。
- 侧边栏。
- 路径栏。
- 面包屑。
- 文件区。
- 状态与任务反馈区。

体验要求：

- 浅色、深色、跟随系统主题保持统一。
- 动效轻量，服务于状态变化和空间关系，不追求炫技。
- 图标、间距、右键菜单、空状态、错误状态保持一致。
- 文件操作必须有清晰进度、结果和错误反馈。
- 冲突处理弹窗支持批量应用。
- 搜索和过滤应即时反馈。
- 缩略图渐进加载，不阻塞列表浏览。

## 7. 明确不做

V1 不做：

- FTP / SFTP。
- 网盘同步。
- Git 状态。
- 插件系统。
- 文件标签。
- 全文索引。
- AI 文件整理。
- 压缩包浏览。
- 高级批量重命名规则。
- 内置预览器。
- 内置终端。
- 局域网共享。
- 跨设备同步。
- 标签分组、会话保存、标签页同步或复杂工作区恢复。

## 9. 成功标准

V1 成功标准：

| 指标 | 标准 |
| --- | --- |
| 启动速度 | <= 800ms |
| 大目录可交互 | <= 500ms |
| 滚动性能 | 稳定 60fps |
| 前台体感 | 后台任务不影响当前可见 UI 的流畅性 |
| 文件操作失败率 | 极低，且可恢复 |
| UI 一致性 | 所有页面统一设计语言 |
| 崩溃率 | 内测阶段低于 1% |
| 可替代性 | 用户能日常替代 Windows 资源管理器 |

## 10. 技术假设

技术假设用于后续 CDD、架构设计和实现规划，不在 PRD 阶段锁死细节：

- 后端/核心：Rust。
- 前端/UI：React。
- 桌面壳：优先考虑 Tauri。
- 目标系统：Windows 10 / 11。

关键实现方向：

- Rust 负责文件系统访问、索引、缩略图、复制任务和后台 IO。
- React 负责 UI 状态和渲染。
- 文件操作任务队列化。
- 前端列表虚拟化。
- 后台线程池处理 IO。
- 后台任务必须服从 UI 优先调度原则。

## 11. 验收口径

V1 验收时必须覆盖以下闭环：

1. 打开应用后进入默认目录。
2. 使用侧边栏、路径栏、面包屑和标签页切换多个目录。
3. 在普通目录和 1 万文件目录中浏览、滚动、排序和过滤。
4. 搜索当前目录，并在递归开关下找到目标文件。
5. 执行新建文件夹、重命名、删除、复制、剪切、粘贴、拖拽移动和拖拽复制。
6. 触发文件冲突，并验证替换、跳过、保留两者、应用于全部。
7. 在后台扫描、缩略图生成或文件操作进行中，验证前台 UI 仍可流畅滚动、选择、右键、路径跳转和切换标签。
8. 切换浅色、深色、跟随系统主题，并验证基础设置生效。
