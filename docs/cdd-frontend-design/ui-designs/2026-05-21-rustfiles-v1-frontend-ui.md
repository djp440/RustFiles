# RustFiles V1 前端 UI 设计文档

## 1. 背景与输入

- PRD：`docs/cdd-brainstorming/prds/2026-05-21-rustfiles-v1-prd.md`
- CDD 规格文档：`docs/cdd-specification/specs/2026-05-21-rustfiles-v1-cdd.md`
- 架构设计文档：`docs/cdd-architecture-design/architectures/2026-05-21-rustfiles-v1-architecture.md`
- 风格输入：`cdd-frontend-design/style/apple-ui-ai-frontend-guide.md`
- 现有设计系统 / 品牌材料：暂无。

已批准 UI 方向：

- 采用 Apple-like Liquid Workbench：Apple 式清晰、内容优先、层级、直接操作感和反馈可恢复，结合液态玻璃 / 毛玻璃 / Acrylic / Mica 材质。
- 全界面必须保持 Apple 式近乎强迫症的统一美感：同一材质语法、圆角、边框、阴影、间距、图标、动效和状态反馈，不允许局部风格漂移。
- 玻璃材质必须兼顾可读性和性能；文件列表、详情表格和关键文字不能因为透明度牺牲清晰度。

## 2. 目标用户与场景

目标用户：

- Windows 10 / 11 上希望日常替代资源管理器的本地文件高频用户。
- 开发者、内容创作者、办公用户和大量文件管理用户。

关键场景：

- 快速进入常用目录。
- 在多标签之间切换不同工作路径。
- 在大目录中滚动、搜索、过滤和排序。
- 执行复制、剪切、粘贴、拖拽、删除、重命名和冲突处理。
- 在后台任务进行时继续浏览和操作当前目录。

使用环境：

- 桌面优先，目标系统为 Windows 10 / 11。
- 支持浅色、深色、跟随系统主题。
- 支持窗口缩放到紧凑宽度，但 V1 不以移动端为目标。

## 3. 设计目标与非目标

### 3.1 设计目标

- 让用户一眼知道当前路径、当前标签、可执行操作和后台任务状态。
- 让文件内容成为主角，UI 材质退到内容之后。
- 用液态玻璃材质建立层级，而不是制造噪音。
- 在高信息密度下保持清爽、统一、可长期使用。
- 所有危险操作都有明确语义、确认和可见反馈。
- 所有加载态、空态、错误态、冲突态和部分完成态都有一致模式。
- 支撑键盘、鼠标和触控板高频操作。

### 3.2 非目标

- 不做营销式首页或装饰性 hero。
- 不做大面积彩虹渐变、赛博霓虹、重玻璃拟态或卡片墙。
- 不为了玻璃效果牺牲列表可读性和滚动性能。
- 不设计移动端完整文件管理体验。
- 不设计复杂标签页工作区、标签分组或跨设备同步界面。
- 不设计 V1.1 视频/PDF 缩略图或 V2 索引/插件界面。

## 4. 视觉方向

### 4.1 设计气质

RustFiles 的界面气质是“安静、精确、清爽、有层级的液态文件工作台”。

关键词：

- 克制。
- 清晰。
- 统一。
- 高密度但不拥挤。
- 内容优先。
- 轻盈但可信。
- 有直接操作感。
- 像真实产品而不是概念 Demo。

### 4.2 记忆点

用户记住 RustFiles 的方式应该是：

> 一个像液态玻璃一样轻盈、但文件操作又非常稳定可信的 Windows 文件管理器。

主界面记忆点：

- 窗口底层有 Acrylic / Mica / Liquid Glass 气质。
- 标签栏、侧边栏、路径栏、任务面板共享统一玻璃材质。
- 文件列表是清晰、稳定、高性能的内容层，不被玻璃干扰。

### 4.3 材质原则

液态玻璃是底层材质系统，不是装饰特效。

材质层级：

1. **Window Backdrop**：窗口级 Acrylic / Mica / Blur，形成整体背景氛围。
2. **Chrome Glass**：标签栏、路径栏、侧边栏、工具栏，使用半透明、模糊、轻边框。
3. **Content Surface**：文件列表和详情区，使用更实的半透明 surface，优先保证文字可读。
4. **Floating Glass**：右键菜单、命令菜单、冲突弹窗、属性面板、任务详情，使用更强层级、阴影和遮罩。
5. **Solid Safety Surface**：永久删除、覆盖、部分完成等高风险状态使用更实的背景，不允许过度透明。

强制规则：

- 不允许每个区域随意使用不同透明度、圆角和阴影。
- 半透明层上的正文对比度必须合格。
- 大目录滚动期间可以降低非关键玻璃模糊强度。
- 必须支持降低透明度模式和不支持 `backdrop-filter` 的 fallback。
- 暗色模式不能简单反色，必须重新定义 surface、border 和 text token。

## 5. 页面 / 视图清单

### 5.1 主文件管理视图

- 目标：完成 V1 的浏览、导航、搜索、排序、过滤、多选和文件操作闭环。
- 关键内容：标签栏、侧边栏、路径栏、面包屑、工具栏、文件列表、状态栏、任务反馈。
- 关键操作：进入目录、返回/前进、切换标签、搜索、排序、过滤、选择、右键、拖拽、复制/移动/删除。

### 5.2 设置视图

- 目标：配置基础偏好。
- 关键内容：默认启动路径、默认视图、隐藏文件、扩展名、主题、排序偏好、缩略图、动效、降低透明度。
- 关键操作：切换设置、保存、恢复默认。

### 5.3 冲突处理弹窗

- 目标：在复制/移动冲突时让用户安全决策。
- 关键内容：源文件、目标文件、时间、大小、缩略图或图标、操作选项。
- 关键操作：替换、跳过、保留两者、应用于全部、取消任务。

### 5.4 文件操作任务面板

- 目标：显示后台任务进度、错误、冲突、取消和部分完成。
- 关键内容：任务列表、进度、速度、剩余数量、当前状态、错误详情。
- 关键操作：查看详情、取消、重试、打开目标位置。

### 5.5 属性 / 信息面板

- 目标：展示文件或文件夹基础属性。
- 关键内容：名称、路径、类型、大小、修改时间、创建时间、权限提示。
- 关键操作：复制路径、打开所在位置、关闭。

## 6. 信息架构与页面结构

### 6.1 主界面布局

桌面主界面从上到下：

1. **Title / Tab Region**：窗口控制区、标签页、新建标签按钮。
2. **Navigation Bar**：返回、前进、地址栏、面包屑、搜索入口、视图切换。
3. **Workspace**：
   - 左侧 Sidebar：快速访问、常用目录、此电脑、磁盘。
   - 右侧 Content：文件列表 / 图标网格 / 详细信息表格。
4. **Status / Task Region**：选中数量、总数、后台任务摘要、错误或同步状态。

布局规则：

- 左侧侧边栏宽度默认 220-260px，可折叠。
- 内容区占据视觉重心。
- 标签栏和路径栏高度稳定，不能因路径变长或搜索状态改变而跳动。
- 文件列表行高、图标网格尺寸、详情列宽必须使用稳定 token。
- 主界面避免卡片套卡片；只有重复项、浮层、弹窗和任务项可以卡片化。

### 6.2 标签页结构

- 标签页位于顶部，视觉上属于窗口 chrome。
- 当前标签使用更高 surface 和轻微高亮边框。
- 非当前标签降低对比但仍可读。
- 标签关闭按钮只在 hover、focus 或当前标签上突出。
- V0.1 就保留 `TabState` 视觉框架，即使完整多标签在 V0.8 完成。

### 6.3 文件列表结构

三种视图：

- 图标视图：适合图片、视频、素材目录；缩略图优先，名称最多两行。
- 列表视图：适合快速扫文件名；高密度、低装饰。
- 详细信息视图：适合排序比较；列包括名称、修改时间、大小、类型。

文件项规则：

- hover、selected、cut-pending、drag-over、focused、disabled 都必须有区分。
- 选中态不能只靠颜色；应结合背景、边框或勾选标记。
- 剪切态降低不透明度并显示轻量状态标识。
- reparse point / symlink / junction 可以用小角标或辅助图标提示。
- 错误或无权限文件不能让整行布局跳动。

## 7. 组件体系

### 7.1 App Shell

- 职责：承载窗口级背景、主题、透明度偏好、全局布局。
- 变体：浅色、深色、跟随系统、降低透明度。
- 规则：所有子区域必须使用同一 token 系统。

### 7.2 Glass Surface

- 职责：统一液态玻璃材质。
- 变体：
  - `chrome`：标签栏、路径栏、侧边栏。
  - `content`：文件列表背景。
  - `floating`：菜单、弹窗、popover。
  - `solid-safety`：危险确认和高风险错误。
- 规则：组件不得临时写独立透明度；必须使用 token。

### 7.3 Sidebar

- 职责：快速访问、常用目录、磁盘入口。
- 状态：默认、hover、selected、collapsed、loading drives、drive error。
- 规则：图标和文字对齐，选中态与文件列表选中态共享语义。

### 7.4 Path Bar / Breadcrumb

- 职责：路径跳转和层级导航。
- 状态：显示模式、编辑模式、无效路径、加载中。
- 规则：长路径中间折叠；编辑模式有明确 focus ring 和提交/取消反馈。

### 7.5 Toolbar

- 职责：视图切换、排序、过滤、搜索入口、常用操作。
- 规则：图标按钮必须有 tooltip 和 accessible name；危险操作不得紧挨高频普通操作。

### 7.6 File Grid / File List / Details Table

- 职责：展示文件条目和选择状态。
- 状态：loading、empty、error、ready、searching、drag-over、multi-selected。
- 规则：虚拟列表滚动时高度稳定；缩略图加载不能改变 item 尺寸。

### 7.7 Context Menu

- 职责：提供右键操作。
- 内容顺序：打开、在新标签页打开、复制/剪切/粘贴、重命名、删除、属性。
- 规则：危险操作分组并使用 danger 语义；菜单项支持键盘导航。

### 7.8 Dialog / Sheet

- 职责：冲突处理、永久删除确认、错误详情、属性。
- 规则：打开后 focus 到标题或第一个安全操作；危险主操作不能默认聚焦。

### 7.9 Task Panel

- 职责：后台任务可见性。
- 状态：queued、validating、running、waiting_for_conflict_decision、cancelling、cancelled、completed、failed、partially_completed。
- 规则：状态文案和图标必须与 Rust 任务状态机一致。

### 7.10 Settings

- 职责：基础偏好设置。
- 规则：使用表单控件而不是自定义花哨交互；每项说明短而清楚。

## 8. 状态设计

### 8.1 加载态

- 普通目录：路径栏显示轻量 loading，文件区使用骨架行或渐进填充。
- 大目录：先显示首批可交互内容，再渐进补充总数、缩略图和缓存状态。
- 缩略图：固定尺寸占位，不改变布局。
- 搜索：搜索框显示进行中状态，结果区可增量出现。

### 8.2 空态

- 空文件夹：安静显示“此文件夹为空”，提供新建文件夹入口。
- 搜索无结果：显示搜索词和清除搜索操作。
- 侧边栏磁盘加载失败：显示可重试状态。

### 8.3 错误态

- 权限不足：说明无法访问，并提供返回上级或重试。
- 路径不存在：说明项目已移动或删除，并提供刷新。
- 回收站不可用：说明删除未执行，不得自动永久删除。
- 缩略图失败：显示文件类型图标，不显示红色大错误。
- 测试根目录 guard 拒绝：开发/测试模式下明确显示越界原因。

### 8.4 成功态

- 新建、重命名、复制、移动、删除成功后使用轻量 toast 或任务完成状态。
- 成功反馈不遮挡文件列表。
- 批量操作成功应显示数量摘要。

### 8.5 危险与部分完成态

- 永久删除确认使用 `solid-safety` surface，明确列出影响对象数量。
- 覆盖冲突必须并排展示源/目标关键信息。
- `partially_completed` 必须展示已完成、未完成、未知状态，并提供打开目标位置或查看详情。

## 9. 交互与反馈规则

### 9.1 直接操作

- 单击选择，双击打开。
- 拖拽时文件项产生 lift feedback，目标目录出现明确 drop indicator。
- 同卷拖拽默认移动，跨卷拖拽默认复制；UI 必须显示当前语义。
- 剪切项保持可见但降低不透明度，并有轻量角标。

### 9.2 搜索与过滤

- 搜索输入即时反馈。
- 递归搜索开启时显示范围提示。
- 搜索结果打开所在位置时，如果项目不存在，显示可恢复错误并刷新目录。

### 9.3 冲突处理

- 冲突弹窗默认不选择危险操作。
- “应用于全部”是独立 checkbox/toggle，不与操作按钮混合。
- 保留两者需要显示将生成的新名称。

### 9.4 动效原则

- 动效必须解释层级、状态和空间关系。
- Tab 切换：180-260ms。
- Modal / Sheet：260-420ms。
- Hover：120-180ms。
- Press：80-120ms。
- Reduced Motion 下保留 opacity 反馈，移除大位移和弹性。

### 9.5 反馈节奏

- 100ms 内可完成的操作不强制显示 loading。
- 超过 300ms 显示轻量进行中状态。
- 超过 1s 的任务进入任务面板或显示进度。
- 后台任务不得用阻塞 modal，除非需要用户处理冲突或危险确认。

## 10. 视觉语言规范

### 10.1 Typography

字体栈：

```css
font-family:
  ui-sans-serif,
  -apple-system,
  BlinkMacSystemFont,
  "SF Pro Text",
  "SF Pro Display",
  "Segoe UI Variable",
  "Segoe UI",
  "Helvetica Neue",
  Arial,
  sans-serif;
```

字号建议：

- 文件名：13-14px，中文不得低于 13px。
- 详情列：12-13px。
- 路径栏：13-14px。
- 设置正文：14-15px。
- 弹窗标题：18-22px。

规则：

- 不使用负 letter spacing。
- 数字列使用 tabular numbers。
- 灰色说明文字不能过浅。

### 10.2 Color

必须使用语义 token：

- `bg.window`
- `surface.chrome`
- `surface.content`
- `surface.floating`
- `surface.solidSafety`
- `text.primary`
- `text.secondary`
- `text.tertiary`
- `border.subtle`
- `accent.primary`
- `danger`
- `warning`
- `success`
- `selection`
- `focus`

强调色：

- 默认使用 Apple-like 蓝色作为主强调色。
- 危险、警告、成功只表达状态，不抢主色角色。

### 10.3 Liquid Glass Token

建议 token：

- `glass.chrome.blur: 24px`
- `glass.chrome.opacity.light: 0.68`
- `glass.chrome.opacity.dark: 0.54`
- `glass.content.blur: 12px`
- `glass.content.opacity.light: 0.82`
- `glass.content.opacity.dark: 0.72`
- `glass.floating.blur: 32px`
- `glass.border.light: rgba(255,255,255,0.42)`
- `glass.border.dark: rgba(255,255,255,0.12)`

Fallback：

- 不支持 `backdrop-filter` 时使用接近实色 surface。
- 降低透明度模式中关闭 blur，使用 solid surface。
- 滚动性能不足时可临时降低 content 区域 blur。

### 10.4 Spacing

基础间距：

- 4px、8px、12px、16px、20px、24px、32px。

桌面布局：

- 标签栏高度：36-44px。
- 路径栏高度：40-48px。
- 侧边栏 item：32-36px。
- 详情列表行高：32-40px。
- 图标网格 cell：96-132px，缩略图尺寸固定。

### 10.5 Borders / Shadows / Radius

- 小控件圆角：8-10px。
- 输入框/按钮圆角：10-12px。
- 浮层圆角：16-20px。
- 弹窗圆角：20-28px。
- 文件 item 圆角：8-12px。

阴影：

- 默认内容层主要用边框和材质区分。
- 浮层和弹窗使用柔和阴影。
- 暗色模式用边框和 surface 层级，不依赖浓黑阴影。

## 11. 响应式与可访问性

### 11.1 断点策略

- Compact：小于 720px，侧边栏默认折叠，工具栏收进菜单。
- Medium：720-1100px，侧边栏可窄版显示，文件列表保持主导。
- Expanded：1100px 以上，完整侧边栏、路径栏、任务区。

V1 桌面优先，不以移动触控为主，但窗口缩小时不能重叠或文字溢出。

### 11.2 键盘

必须支持：

- Tab / Shift+Tab 焦点移动。
- 方向键移动文件焦点。
- Enter 打开。
- Space 选择。
- Ctrl/Cmd 类快捷键在 Windows 上按 Ctrl 语义设计。
- Escape 关闭菜单、弹窗或取消搜索焦点。
- 右键菜单支持键盘导航。

### 11.3 Focus 与读屏

- 所有 icon-only button 必须有 accessible name。
- focus ring 明确可见，不能被玻璃背景吞掉。
- Dialog 打开后管理焦点。
- 错误信息与相关控件关联。
- 任务状态变化谨慎使用 aria-live，避免频繁刷屏。

### 11.4 Reduced Motion / Transparency

- 支持 `prefers-reduced-motion`。
- 支持降低透明度设置。
- 降低透明度后仍保持层级，不只是粗暴变成纯白/纯黑。

## 12. 前端 e2e 验收设计

### 12.1 推荐验证工具

- 开发期 Web UI：Playwright 或可用浏览器工具。
- 桌面期：Tauri 窗口自动化 / 可用浏览器工具结合截图和文件系统断言。
- 视觉验收：桌面宽度、紧凑宽度、浅色、深色、降低透明度模式截图。

### 12.2 关键用户路径

路径 1：浏览与导航

- 起点：应用启动进入默认目录。
- 操作：点击侧边栏、进入文件夹、返回、前进、编辑路径、点击面包屑。
- 断言：当前路径、列表内容、选中侧边栏和面包屑状态正确；无布局跳动。

路径 2：视图、排序、过滤

- 操作：切换图标/列表/详细信息视图，按名称/时间/大小/类型排序，应用文件类型过滤。
- 断言：视图密度、列、选中态、空态和排序反馈正确。

路径 3：搜索

- 操作：输入搜索词、开启递归、打开所在位置、清除搜索。
- 断言：搜索进行中、结果、无结果、项目不存在错误态正确。

路径 4：文件操作反馈

- 操作：新建文件夹、重命名、删除到回收站、永久删除确认、复制/移动冲突。
- 断言：危险确认、冲突处理、任务面板、成功/失败 toast 可见且不阻塞非必要 UI。

路径 5：多标签与后台任务

- 操作：打开多个标签，后台标签执行任务，前台标签滚动和选择。
- 断言：前台 UI 流畅，后台任务状态可见，状态不串标签。

路径 6：玻璃与可读性

- 操作：切换浅色、深色、降低透明度；在大目录滚动。
- 断言：文本可读，focus 可见，列表不掉帧，材质风格一致。

### 12.3 页面状态验收

- 加载态：目录加载、缩略图加载、搜索加载。
- 空态：空目录、搜索无结果。
- 错误态：权限不足、路径不存在、回收站失败、缓存不可写。
- 成功态：操作完成、设置保存。
- 危险态：永久删除、覆盖、部分完成。

### 12.4 响应式验收

- Compact：侧边栏折叠，工具栏不溢出，路径可编辑。
- Medium：侧边栏窄版，文件列表仍可读。
- Expanded：完整布局，任务区和侧边栏层级清楚。

### 12.5 可访问性验收

- 全键盘完成浏览、搜索、右键菜单和弹窗关闭。
- focus ring 在玻璃和实色 surface 上都可见。
- 图标按钮具备 accessible name。
- Reduced Motion 和降低透明度生效。
- 文本放大后按钮和列表不重叠。

### 12.6 不应通过的反例

- 毛玻璃导致文件名或详情列不可读。
- 不同区域使用不一致的圆角、透明度或阴影。
- 大目录滚动时玻璃材质导致明显掉帧。
- 危险操作默认聚焦在破坏性按钮。
- 错误态只显示“失败”，没有原因和下一步。
- 图标按钮没有 tooltip 或 accessible name。
