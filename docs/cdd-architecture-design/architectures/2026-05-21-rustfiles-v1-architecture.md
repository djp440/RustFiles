# RustFiles V1 架构设计文档

## 1. 背景与输入

- PRD：`docs/cdd-brainstorming/prds/2026-05-21-rustfiles-v1-prd.md`
- CDD 规格文档：`docs/cdd-specification/specs/2026-05-21-rustfiles-v1-cdd.md`
- 当前代码库：只有文档、LICENSE、README 和 git 配置，尚无 Rust、React 或 Tauri 代码结构。
- 用户确认的关键架构原则：
  - 采用“Rust 后端唯一写入者”原则。
  - React 前端只发起意图和展示状态，不直接执行真实文件写入、删除、移动、冲突决策执行或系统集成。
  - 调试阶段允许前端和后端分开运行。
  - 发布阶段必须至少能打包为一个可直接启动的桌面二进制入口。

## 2. 目标与非目标

### 2.1 架构目标

- 支撑 V1 的本地文件浏览、多标签、搜索、排序、过滤、缩略图、基础设置和文件操作闭环。
- 明确 Rust Core、React UI、Tauri 桌面壳之间的职责边界。
- 将所有真实文件系统副作用收敛到 Rust 后端，便于 R4 风险审核、测试隔离和失败恢复。
- 支撑 UI 优先调度：前台输入、滚动、动画、标签切换和路径跳转优先于后台扫描、缩略图、搜索和文件操作。
- 支撑开发期分离调试与发布期单二进制启动。
- 为后续实现计划提供可直接拆任务的模块边界、接口契约和验证策略。

### 2.2 非目标

- 不设计云盘同步、远程文件协议、插件系统、全文索引、AI 文件整理、压缩包浏览、内置终端或内置预览器。
- 不设计复杂标签页工作区、标签分组、会话同步或跨窗口拖拽标签。
- 不引入全盘索引或数据库型目录索引作为 V1 基础。
- 不在本阶段确定具体 UI 视觉稿、组件样式或像素级布局。
- 不在本阶段拆分具体实现任务或编写代码。

## 3. 系统上下文

### 3.1 参与者

- 用户：通过桌面 UI 浏览和操作本地文件。
- React UI：负责展示、交互、本地 UI 状态和用户意图表达。
- Tauri Shell：负责桌面窗口、前后端桥接、打包和系统权限边界。
- Rust Core：负责文件系统访问、后台任务、系统集成、缩略图、设置和观测。
- Windows 系统：提供文件系统、回收站、默认应用、终端、属性、磁盘分区和主题信息。

### 3.2 关键依赖

- Windows 10 / 11 文件系统 API。
- Windows 回收站能力。
- Windows 默认应用打开能力。
- Windows 终端或系统 shell 打开能力。
- Tauri command / event 桥接。
- React 虚拟列表与前端状态管理。
- Rust 异步运行时或线程池。
- 本地配置文件和缩略图缓存目录。

## 4. 组件与模块划分

### 4.1 React UI 层

- 职责：
  - 渲染主窗口、标签栏、侧边栏、路径栏、面包屑、文件列表、右键菜单、设置页和任务反馈区。
  - 管理纯 UI 状态：选中项、焦点、展开状态、弹窗显示、当前输入文本、视图模式临时展示。
  - 维护标签页视图状态：当前路径、历史栈、视图模式、排序、过滤、搜索输入和滚动位置。
  - 发起结构化命令请求，并订阅 Rust 事件。
- 边界：
  - 不直接写入、删除、移动或覆盖真实文件。
  - 不直接调用系统回收站、默认应用、终端或属性窗口。
  - 不自行判断破坏性操作是否已经完成，只展示 Rust 返回的任务状态。
- 依赖：
  - Tauri command。
  - Tauri event。
  - 虚拟列表库。

### 4.2 Tauri Shell 层

- 职责：
  - 管理桌面窗口生命周期、菜单和前后端桥接。
  - 提供发布期单二进制入口。
  - 开发期支持前端 dev server 与 Rust/Tauri 后端分开调试。
  - 控制前端可调用命令白名单和权限边界。
- 边界：
  - 不承载业务逻辑。
  - 不让前端绕过 Rust Core 直接访问危险系统能力。

### 4.3 Rust Core 总入口

- 职责：
  - 聚合文件浏览、任务队列、调度、搜索、缩略图、设置、系统集成和观测模块。
  - 为 Tauri 暴露稳定 command。
  - 向前端发布增量事件。
- 边界：
  - 所有真实文件系统副作用必须经过 Rust Core。
  - 所有命令入参必须经过路径规范化、权限边界和测试模式检查。

### 4.4 `fs` 文件浏览模块

- 职责：
  - 枚举目录。
  - 读取文件元数据。
  - 应用排序、过滤和分页。
  - 识别文件类型与基础图标类型。
  - 监听或刷新目录变化。
- 边界：
  - 只负责读路径。
  - 不执行写入、删除、移动或复制。

### 4.5 `tasks` 文件操作任务模块

- 职责：
  - 统一承载新建文件夹、重命名、删除到回收站、永久删除、复制、剪切、粘贴、拖拽移动和拖拽复制。
  - 为每个操作创建任务 id、阶段、进度、错误、取消和恢复状态。
  - 在冲突时暂停任务，并等待前端提交用户选择。
  - 将任务事件推送给前端。
- 边界：
  - 是真实文件写入副作用的唯一入口。
  - 不直接处理 UI 展示。
  - 不在缺少二次确认时执行永久删除。

### 4.6 `scheduler` UI 优先调度模块

- 职责：
  - 管理后台任务优先级。
  - 识别前台标签页、可视区域和用户交互期。
  - 在用户滚动、选择、路径跳转或标签切换时降低非关键后台任务优先级。
  - 支持取消或延迟非可视缩略图任务。
- 边界：
  - 不改变文件操作的正确性语义。
  - 只能影响任务执行顺序、并发度和让步策略。

### 4.7 `search` 搜索模块

- 职责：
  - 当前目录文件名搜索。
  - 实时过滤。
  - 可选递归搜索。
  - 返回搜索结果和所在位置。
- 边界：
  - 不做全文搜索。
  - 不做全盘索引。
  - 递归搜索必须可取消，并服从 UI 优先调度。

### 4.8 `thumbnails` 缩略图模块

- 职责：
  - 图片缩略图生成。
  - 文件夹图标和常见文件类型图标解析。
  - 本地缓存、缓存命中和失效。
  - 可视区域优先加载。
- 边界：
  - V1 不要求视频或 PDF 缩略图。
  - 缩略图失败不能阻塞文件列表。

### 4.9 `settings` 设置模块

- 职责：
  - 持久化默认启动路径、默认视图、隐藏文件、扩展名、主题、排序偏好、缩略图开关和动效开关。
  - 向前端提供初始化设置和更新确认。
- 边界：
  - 设置写入失败必须可见。
  - 不存储敏感凭据。

### 4.10 `system` Windows 集成模块

- 职责：
  - 删除到回收站。
  - 默认应用打开。
  - 终端打开当前目录。
  - 显示属性。
  - 读取常用位置和磁盘分区。
  - 读取系统主题偏好。
- 边界：
  - 所有系统集成失败必须返回结构化错误。
  - 任何需要管理员权限或修改系统配置的能力必须单独升级审查。

### 4.11 `observability` 观测模块

- 职责：
  - 结构化日志。
  - 性能计时。
  - 任务审计事件。
  - 崩溃日志入口。
- 边界：
  - 默认不记录完整敏感路径到可共享日志；需要脱敏或仅本地保存。
  - 性能和错误证据必须能支持 CDD 验收。

## 5. 数据流与状态归属

### 5.1 浏览读路径

1. React 当前标签页发起 `list_directory` 请求，包含路径、排序、过滤、分页窗口和是否显示隐藏文件。
2. Tauri 将请求传给 Rust Core。
3. Rust `fs` 规范化路径并枚举目录。
4. Rust 返回首批文件快照和分页信息。
5. React 渲染虚拟列表。
6. Rust 对可视区域触发缩略图任务，并通过事件增量返回缩略图状态。

状态归属：

- 当前标签路径、历史栈、视图模式、排序、过滤和滚动位置归 React 标签状态。
- 文件系统真实状态归 Windows 文件系统。
- 目录枚举结果是 Rust 生成的只读快照，React 可缓存但必须允许刷新。

### 5.2 搜索路径

1. React 根据搜索输入发起 `search_directory` 或更新搜索任务。
2. Rust `search` 创建可取消搜索任务。
3. 非递归搜索优先使用当前目录快照；递归搜索进入后台任务。
4. Rust 通过事件推送增量结果、完成或错误。
5. React 显示结果，并通过“打开所在位置”触发导航意图。

状态归属：

- 搜索输入和展示状态归 React。
- 搜索任务执行、取消和错误归 Rust。

### 5.3 文件操作写路径

1. React 发起结构化操作请求，例如复制、移动、删除或重命名。
2. Rust `tasks` 校验路径、确认操作类型、创建任务 id。
3. Rust 执行任务并推送进度事件。
4. 如发生冲突，任务进入 `waiting_for_conflict_decision`。
5. React 展示冲突弹窗并提交用户选择。
6. Rust 继续执行或跳过，并最终返回完成、取消或失败状态。
7. React 根据任务结果刷新相关目录快照。

状态归属：

- 真实文件操作阶段、进度、错误、冲突和取消状态归 Rust。
- 用户选择和弹窗展示归 React。
- 操作审计归 Rust 本地日志。

### 5.4 设置路径

1. 启动时 Rust `settings` 读取配置。
2. React 初始化主题、视图偏好和默认目录。
3. 用户修改设置时，React 发起 `update_settings`。
4. Rust 写入配置并返回确认。
5. React 只在 Rust 确认后展示持久化成功状态。

## 6. 接口与契约

### 6.1 Command 契约类别

- 浏览类：
  - `list_directory(request) -> DirectoryPage`
  - `get_sidebar_roots() -> SidebarRoots`
  - `get_drives() -> DriveList`
- 搜索类：
  - `start_search(request) -> TaskId`
  - `cancel_task(task_id) -> TaskStatus`
- 文件操作类：
  - `create_folder(request) -> TaskId`
  - `rename_item(request) -> TaskId`
  - `delete_to_recycle_bin(request) -> TaskId`
  - `delete_permanently(request) -> TaskId`
  - `copy_items(request) -> TaskId`
  - `move_items(request) -> TaskId`
  - `resolve_conflict(request) -> TaskStatus`
- 缩略图类：
  - `request_thumbnails(request) -> ThumbnailBatchStatus`
  - `cancel_thumbnail_requests(request) -> Ack`
- 设置类：
  - `get_settings() -> Settings`
  - `update_settings(request) -> Settings`
- 系统集成类：
  - `open_with_default_app(path) -> Ack`
  - `open_terminal(path) -> Ack`
  - `show_properties(path) -> Ack`

### 6.2 Event 契约类别

- `directory_changed`：目录内容变化或刷新建议。
- `task_progress`：任务进度、速度、当前阶段。
- `task_conflict`：任务暂停并等待冲突选择。
- `task_completed`：任务完成。
- `task_failed`：任务失败，包含可恢复信息。
- `thumbnail_ready`：缩略图可用。
- `search_result_batch`：搜索结果增量。
- `performance_sample`：关键性能采样。

### 6.3 关键数据结构

- `FileEntry`：路径、名称、类型、大小、修改时间、是否隐藏、是否文件夹、图标类型、缩略图状态。
- `DirectoryPage`：路径、条目列表、总数、分页窗口、排序、过滤、快照版本。
- `TabState`：标签 id、路径、历史栈、视图模式、排序、过滤、搜索状态、滚动位置。
- `FileTask`：任务 id、类型、源、目标、阶段、进度、错误、冲突、是否可取消。
- `ConflictDecision`：替换、跳过、保留两者、应用于全部。
- `Settings`：默认启动路径、默认视图、显示隐藏文件、显示扩展名、主题、排序偏好、缩略图开关、动效开关。

## 7. 存储与基础设施

### 7.1 持久化

- 设置：使用本地配置文件或 Tauri 推荐应用数据目录。
- 缩略图缓存：使用应用缓存目录，支持清理、失效和测试专用目录覆盖。
- 任务历史：V1 可以只保留运行时任务状态；若记录历史，只记录必要审计信息和脱敏路径。
- 崩溃日志：V1.0 打磨阶段加入本地日志文件。

### 7.2 队列与调度

- 文件操作任务队列：
  - 承载真实写操作。
  - 支持取消、暂停、冲突等待和失败状态。
  - 默认限制并发，避免 IO 争抢拖慢 UI。
- 缩略图任务队列：
  - 可视区域优先。
  - 快速滚动时取消或延后非可视任务。
- 搜索任务队列：
  - 支持递归搜索取消。
  - 优先级低于当前可见 UI 交互。

### 7.3 运行模式

- 开发模式：
  - React dev server 和 Tauri/Rust 后端可分开运行，便于调试 UI、命令和事件。
  - 允许开启更详细日志、测试缓存目录和夹具根目录。
- 发布模式：
  - 至少提供一个可直接启动的桌面二进制入口。
  - 前端资源应随桌面应用打包。
  - 用户启动应用不需要单独启动前端服务或后端服务。

## 8. 失败处理与回滚

### 8.1 可预期失败

- 路径不存在。
- 权限不足。
- 文件被占用。
- 目标已存在。
- 跨磁盘移动失败。
- 回收站不可用。
- 缩略图解码失败。
- 缓存目录不可写。
- 设置写入失败。
- 应用关闭或任务中断。

### 8.2 处理策略

- 所有失败返回结构化错误：错误码、用户可读消息、是否可重试、是否需要刷新目录。
- 文件操作任务必须记录当前阶段，避免 UI 不知道真实状态。
- 冲突处理必须暂停任务，不得在无用户选择时默认覆盖。
- 永久删除必须由前端显示二次确认，并由 Rust 校验确认令牌或确认字段后执行。
- 跨磁盘移动应按复制后删除的安全流程处理；源删除前必须确认目标写入成功。
- 后台任务失败不得污染当前标签页的路径、选择和视图状态。

### 8.3 回滚边界

- 新建文件夹失败：不产生残留或返回已存在状态。
- 重命名失败：保持原路径可见。
- 复制失败：保留已复制部分并明确提示，后续实现计划需决定是否支持清理已复制部分。
- 移动失败：优先保证源文件不丢失。
- 删除到回收站失败：不得降级为永久删除。
- 永久删除失败：显示失败状态，不得假装完成。

## 9. 观测性

### 9.1 日志

- 应记录：任务创建、任务阶段变化、冲突、取消、失败、完成、目录枚举耗时、缩略图耗时、搜索耗时。
- 高风险文件操作日志默认本地保存，路径需脱敏或提供隐私模式。
- 开发模式允许更详细日志。

### 9.2 指标

- 冷启动耗时。
- 普通目录打开耗时。
- 1 万文件目录首屏可交互时间。
- 滚动帧时间或掉帧采样。
- 后台任务期间交互延迟。
- 缩略图缓存命中率。
- 文件操作成功率和失败类型。

### 9.3 告警与错误反馈

- V1 本地桌面应用不需要远程告警作为首发必要条件。
- UI 必须展示文件操作失败、设置写入失败、回收站失败和权限不足。
- 崩溃日志在 V1.0 商业首发打磨阶段纳入。

## 10. 系统级 e2e 验证设计

### 10.1 验证目标

- 验证 V1 最小闭环在真实桌面应用中可完成。
- 验证 Rust 后端唯一写入者边界没有被前端绕过。
- 验证后台任务压力下当前 UI 仍保持流畅。
- 验证所有破坏性文件操作只作用于隔离测试目录。

### 10.2 推荐工具

- Rust 测试：`cargo test`，覆盖路径规范化、任务状态机、冲突处理、文件操作策略。
- 前端测试：TypeScript 类型检查、组件/状态测试。
- 桌面 e2e：后续可使用浏览器/桌面自动化工具驱动 Tauri 窗口，结合文件系统断言。
- 性能验证：应用内性能采样日志 + 自动化脚本生成大目录夹具。

### 10.3 启动方式

- 开发期：
  - 启动 React dev server。
  - 启动 Tauri dev 进程。
  - 使用测试专用配置目录、缓存目录和夹具根目录。
- 发布期：
  - 启动单个桌面二进制。
  - 使用打包后的前端资源。

### 10.4 测试数据与环境隔离

- 所有 e2e 文件操作使用测试根目录。
- 必须生成：
  - 小目录。
  - 1 万文件大目录。
  - 冲突源/目标目录。
  - 图片与损坏图片目录。
  - 深层递归搜索目录。
  - 权限或占用场景。
- 禁止在真实桌面、下载、文档、图片、视频、音乐目录执行删除、移动或覆盖测试。

### 10.5 阶段级验收标准

- V0.1-V0.2：导航、视图、排序和过滤可用，且没有真实写操作。
- V0.3：大目录虚拟列表和 UI 优先调度有性能证据。
- V0.4：搜索任务可取消，递归搜索不阻塞 UI。
- V0.5-V0.7：文件操作在隔离真实文件系统中通过成功路径、冲突路径和负面路径验证。
- V0.8：多标签页状态隔离通过，后台标签任务不污染当前标签。
- V0.9：缩略图可视区域优先和缓存验证通过。
- V1.0：设置、主题、错误体验、崩溃日志、自动更新和安装包验证通过。

### 10.6 前端设计分支

该项目对 UI 体验要求明显：现代、精致、克制、高密度但不拥挤，并且目标是日常替代资源管理器。因此架构设计完成后，建议先进入 `cdd-frontend-design`，再进入 `cdd-writing-plans`。

## 11. 安全、权限与性能约束

### 11.1 权限边界

- React 不具备真实文件写入权限。
- Rust Core 是唯一真实文件副作用入口。
- 所有路径入参必须规范化，并拒绝测试模式下越过测试根目录的破坏性操作。
- 永久删除、覆盖和移动必须经过明确用户意图。
- 任何管理员权限、系统配置修改或文件关联注册都不进入 V1 默认路径。

### 11.2 敏感数据

- 日志和任务历史默认不上传。
- 路径、文件名和用户目录属于隐私信息。
- 可共享日志需要脱敏。

### 11.3 性能热点

- 大目录枚举。
- 虚拟列表渲染。
- 图片缩略图生成。
- 递归搜索。
- 复制/移动大文件或大量小文件。
- 多标签页后台任务竞争。

性能策略：

- 首屏优先，非可视区域延后。
- 后台任务可降速，前台 UI 不让步。
- 文件操作正确性优先于吞吐。
- 避免在前端主线程做重计算。

## 12. 关键技术取舍

### 12.1 采用分层任务驱动架构

- 方案：React UI + Tauri Shell + Rust Core 分层，真实写操作进入 Rust 任务队列。
- 取舍原因：最匹配 R4 文件安全风险、UI 优先调度和后续可验证性。
- 不采用简单命令式架构：短期快，但任务状态、取消、冲突和后台优先级会分散。
- 不采用索引优先架构：搜索和大目录能力强，但 V1 不做全盘索引，复杂度过高。

### 12.2 不引入全盘索引数据库

- 方案：V1 目录浏览以实时枚举、快照、分页和当前目录搜索为主。
- 取舍原因：符合 V1 范围，降低一致性和索引维护成本。
- 保留空间：后续 V2 可增加索引服务，但不应污染 V1 核心路径。

### 12.3 设置与缩略图轻量持久化

- 方案：设置和缩略图缓存使用应用数据/缓存目录，不引入重型数据库作为 V1 必需依赖。
- 取舍原因：足够满足 V1，便于清理和测试隔离。

### 12.4 开发期分离，发布期单入口

- 方案：开发期允许 React dev server + Tauri/Rust 后端分开调试；发布期打包为单个桌面二进制入口。
- 取舍原因：兼顾调试效率和用户安装启动体验。

- 具体 Rust crate 选择。
- 具体 React 状态管理库选择。
- 具体 Tauri 配置文件内容。
- 自动更新服务端方案。
- 官网落地页架构。
- V1.1 视频/PDF 缩略图实现。
